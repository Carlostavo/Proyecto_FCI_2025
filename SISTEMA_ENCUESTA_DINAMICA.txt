├─ orden (INT)
├─ activo (BOOLEAN)                 ← Mostrar/Ocultar sin eliminar
├─ condicion_visible_json (JSONB)   ← {"pregunta_id": "...", "valor": "Sí"}

ESCENARIO 1: ADMINISTRADORA CREA UNA ENCUESTA
─────────────────────────────────────────────

1. Administradora va a /diagnostico-admin
   └─ Carga obtenerBloquesDinamicos() → Lee 5 bloques
   
2. Selecciona un bloque (Ej: "Información Base")
   └─ Click en "Agregar Pregunta"
   
3. Llena formulario:
   ├─ Texto: "¿En qué parroquia está tu negocio?"
   ├─ Tipo: "opcion_multiple"
   ├─ Opciones: "Parroquia1\nParroquia2\nParroquia3"
   └─ Requerida: ✓
   
4. Click en "Crear Pregunta"
   └─ Ejecuta crearPreguntaDinamica() → Inserta en BD
   
5. Pregunta aparece en la lista
   └─ Puede editar, ocultar o eliminar


ESCENARIO 2: ADMINISTRADORA CONFIGURA CONDICIONES
──────────────────────────────────────────────────

1. Administradora en /diagnostico-admin
2. En tab "Condiciones de Visibilidad"
3. Selecciona pregunta A: "¿Tienes estrategia de marketing?"
4. Configura que Pregunta B sea visible SOLO si responden "Sí"
5. Ahora Pregunta B se muestra condicionalmente


ESCENARIO 3: EMPRENDEDORA RESPONDE ENCUESTA
─────────────────────────────────────────────

1. Emprendedora va a /diagnostico
2. Ingresa ID de participante (Ej: "PART_001")
3. Componente carga obtenerBloquesDinamicos()
   └─ Ve preguntas que administradora creó
   
4. Responde Bloque 1: "Información Base"
   ├─ Pregunta 1: Selecciona parroquia → guardarRespuestaCSV()
   ├─ Pregunta 2: Escribe sector → guardarRespuestaCSV()
   └─ Click "Siguiente"
   
5. Responde Bloques 2-5 del mismo modo

6. Click "Enviar Encuesta"
   ├─ Guarda últimas respuestas
   ├─ Ejecuta marcarEncuestaEnviada()
   └─ Muestra confirmación


ESCENARIO 4: ADMINISTRADORA EDITA ENCUESTA EN TIEMPO REAL
─────────────────────────────────────────────────────────

1. Administradora realiza cambio: edita pregunta, oculta, elimina
   └─ Ejecuta actualizarPreguntaDinamica() o toggleVisibilidadPregunta()
   
2. Emprendedora recarga /diagnostico
   └─ Carga obtenerBloquesDinamicos() nuevamente
   └─ Ve cambios reflejados automáticamente

3. Historial se registra en encuesta_preguntas_auditoria

✅ ROW LEVEL SECURITY (RLS)
├─ Las tablas principales tienen RLS habilitado
├─ Administradoras ven todo
└─ Respuestas vinculadas a id_participante (no user_id)


✅ VALIDACIÓN EN SERVIDOR
├─ Todas las operaciones en Server Actions
├─ No confía en datos del cliente
└─ Verifica integridad de datos


✅ AUDITORÍA AUTOMÁTICA
├─ Trigger en encuesta_preguntas
├─ Registra: QUÉ, QUIÉN, CUÁNDO, ANTES/DESPUÉS
└─ No se puede "borrar historial"


✅ SIN user_id DE AUTH
├─ Funciona con id_participante (TEXT)
├─ Compatible con datos CSV
└─ No requiere que exista usuario en auth
